# StageHopper

A swipe-based festival companion app for the New Orleans Jazz & Heritage Festival 2026. Preview artists via YouTube, build a personal schedule, and discover lesser-known acts — installable as a PWA on iOS and Android.

**Live at [stagehopper.app](https://stagehopper.app)**

---

## The Story

Last year at the Montreal Jazz Festival I had a frustrating experience: hundreds of acts across a dozen stages, no way to preview who anyone was, and no help deciding where to go next. I'd wander from stage to stage with zero context, missing sets I would have loved.

I wanted to build something better: a tool that lets you *hear* the artists before committing to a stage, surfaces scheduling conflicts before they surprise you, and helps you discover acts you'd never have found otherwise.

This year at Jazz Fest 2026 I built StageHopper in roughly a day using Claude Code, then used it live at the festival — finding bugs, adding features, and deploying fixes in real time using Claude Code's `/remote-control` feature from my phone. The gap between "I found a bug" and "it's fixed on the live site" was often under five minutes.

One of the best features came from a 20-something guy on a porch just outside the festival grounds who asked "why do I have to press play on each video?" — describing TikTok's autoplay UX. That suggestion shipped before I left his porch.

## Features

### Discover
The core view. Shows what's playing **right now**, then **up next** — one timeslot at a time so you're never overwhelmed by the full 630-act schedule.

- **Autoplay video previews** — TikTok-style, videos start as you browse. Toggle ▶/⏸ to silence.
- **Swipe right to add** a set to your schedule; **swipe left to skip**
- **Conflict warnings** — if a set overlaps something already on your schedule, it's flagged inline
- **Artist bios** — sourced from Last.fm where available, AI-generated otherwise
- **Browse by stage** — tap "By stage" to filter the deck to a single stage
- **Search** — tap 🔍 to search across all 577 artists by name

### Mine
Your personal schedule for the selected day.

- All your picks in chronological order
- **Conflict detection** with a ★ top pick recommendation when two sets overlap
- Tap any set to preview the video and remove if needed
- Badge count reflects the active day so you always know how many picks you have

### Official Schedule
A complete **14-stage × time-axis grid** for any day, matching the layout of the paper schedule at the gate.

- Sticky stage headers and time axis as you scroll
- **Long-press any block** to preview that act (video + bio) without leaving the grid
- Tap the preview to add or remove from your schedule
- Color-coded by stage

### General
- **Installable PWA** — works offline after first load, runs full-screen like a native app
- **Notification support** — optional alerts before your scheduled sets
- **Festival and accessibility maps** in the hamburger menu
- **Dark theme** throughout
- **Day tabs** for all 8 festival days (two weekends: Apr 24–26, Apr 30–May 3)

## Data & Video Selection

- **630 performances** across **14 stages** over **8 days**
- **577 unique artists**, each with a YouTube embed and a bio
- Videos selected by `best_yt.sh` + `rerank_yt.js`: searches YouTube via `yt-dlp`, ranks by view count, filters out news/TV channels, and validates embeddability via the YouTube oembed endpoint before writing to `schedule.js`
- Band names are quoted in the search query and stage/city context is appended to avoid matching song titles instead of band names (e.g. "Boyfriend" the band, not the Justin Bieber song)
- Tribute acts (e.g. "A Tribute to John Coltrane featuring Derek Douget…") search by the featured performers, not the tribute subject
- Bios generated by `gen_bios.js`: Last.fm first, Claude Haiku fallback for unknowns

## Files

| File | Purpose |
|------|---------|
| `index.html` | PWA entry point |
| `app.jsx` | Main React UI (Discover, Mine, Official Schedule, Search) |
| `schedule.js` | All performer/stage/time/video data |
| `sw.js` | Service worker — cache versioning for offline support |
| `manifest.json` | PWA manifest |
| `gen_bios.js` | Fetches artist bios from Last.fm + Claude Haiku |
| `best_yt.sh` | Finds best YouTube video for a band name via yt-dlp |
| `rerank_yt.js` | Runs `best_yt.sh` for all artists, validates embeddability, patches `schedule.js` |

## Local Dev

```bash
python3 -m http.server 8001
# open http://localhost:8001
```

React + Babel run entirely in the browser from CDN — no build step required.

## Deploying Updates

1. Edit files and push to `main`
2. GitHub Pages rebuilds automatically
3. Bump `CACHE_VERSION` in `sw.js` to force installed PWAs to pick up the new files
4. If stagehopper.app is behind Cloudflare, purge the cache after pushing

## Install on iPhone

Open [stagehopper.app](https://stagehopper.app) in **Safari** → tap Share → "Add to Home Screen". Launches full-screen, works offline after first load.
